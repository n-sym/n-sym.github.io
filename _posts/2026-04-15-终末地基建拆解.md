笔者前段时间细致地体验了终末地，在此稍微写下一些想法。

### 游戏定位
终末地是一款二次元+策略ARPG+工厂游戏，它做了一种一以贯之的*差异化*，使得玩家在接触时有可能感受到游戏是独特甚至小众的，难以找到同品类游戏替代，最终令它能占据一种独特的生态位，在市面上会比较新奇。因此，也许可以认为引入基建并非主要为了满足硬核工厂类游戏玩家（不过他们也有可能来玩）。正相反，它反倒希望把基建推广给更多原本没怎么接触过类似类型游戏的玩家，从而产生粘性。笔者这么说的信心是基于基建和战斗系统本身都某种程度上比较特殊，并且在社区中部分发言的玩家的气质也间接支持这一点。

### 设计目的

开门见山，笔者认为，基建
- 作为地图探索玩法和战斗、养成玩法之间的填充过渡，更有效地维持游戏的新鲜感
- 减少了重复劳动性的一部分日常收菜时间，转化为集中的基建设计整块时间，更有趣
- 借助蓝图和各类整活玩法促进社区的活力，鼓励轻度的社交

### 核心循环

 经由基建，玩家可获得：

| 类别         | 描述                 | 所属系统  | 最终兑现于 |
| ---------- | ------------------ | ----- | ----- |
| 装备原件       | 可以用于装备制造，提升战斗实力    | 装备制造  | 战斗体验  |
| 工业药品       | 回复血量               | 战斗消耗品 | 战斗体验  |
| 水瓶、工业爆炸物   | 在地图探索中解谜           | 快捷道具  | 探索体验  |
| 调度券        | 用于进行物资调度           | 物资调度  | 多种    |
| 常驻抽卡券（一次性） | 作为整个基建系统提供给玩家的动力之一 | 抽卡    | 抽卡体验  |
| 养成资源（一次性）  | 辅助前期玩家进行养成         | 养成    | 战斗体验  |

其中物资调度的产物可以分为几类：

| 类别         | 描述                 | 所属系统 | 最终兑现于        |
| ---------- | ------------------ | ---- | ------------ |
| 刻写券        | 用于提升基质的词条等级        | 养成   | 战斗体验         |
| 精锻助剂       | 用于提升装备的词条等级        | 装备制造 | 战斗体验         |
| 养成资源（一次性）  | 辅助前期玩家进行养成         | 养成   | 战斗体验         |
| 礼物         | 用于提升干员信赖           | 信赖   | 某种情感需求，战斗体验  |
| 收藏品（一次性）   | 可以摆在帝江号上           | 收藏？  | 某种情感需求       |
| 常驻抽卡券（一次性） | 作为整个基建系统提供给玩家的动力之一 | 抽卡   | 抽卡水位和抽卡得到的道具 |
| 罗盘等        | 搜寻地图中未获取的储藏箱/醚质    | 探索   | 节省时间         |

而玩家需要为基建系统投入这些资源：

| 类别   | 描述                          | 来源                |
| ---- | --------------------------- | ----------------- |
| 工业点数 | 用于解锁工业技能树                   | 地图探索              |
| 矿机点位 | 玩家需要铺设电网并在矿点上建设矿机，获得基建的基础资源 | 地图探索              |
| 水泵点位 | 与上一条类似                      | 地图探索              |
| 据点发展 | 解锁额外的据点以及提升物资调度的产出效率        | 任务                |
| 调度券  | 用于解锁更大的建设面积                 | 战斗（极少部分），主要是基建自给的 |

考虑玩家的兴趣，在探索、基建、养成、战斗间的流转会比普通的探索、养成、战斗更随机化，下图是省略的表示。

![](/assets/post-images/zmd.svg)


在这个过程中，玩家可以不断见到自己的数值成长以及基建据点的壮大，这提供获得感。

对于第一点，在最开始，玩家不能从探索直接转向战斗，必须要进行基建来获得装备，玩家不会过早进入探索-养成的循环。最终，由于养成收益的边际效应明显，基建的产出已经不再有用，且基建本身产出调度券效率有上限，循环会被打破，玩家不再需要回到基建，只需要在乎战斗和养成的循环，此时基建就退场了。基建维持的就是前期的新鲜感。还需要补充的是，终末地是个持续运营游戏，基建的内容也会随版本更新。因此基建是能够隔一段时间就短时间刷一次存在感的。

对于第二点，如果希望类比，整个基建过程实际上相当于**点按**原神的圣遗物副本的“解锁”交互按钮的瞬间，只是整个过程被放得漫长，占据游玩时间，被设计为游戏的主要玩法之一。对于解锁按钮，玩家需要先通过探索找到它，然后凑近按F解锁一个兑现体力的出口。而基建则是玩家需要先通过探索得到工业点数和初级资源，然后搭建一个产线来解锁高效兑现体力的出口。

在这个类比中，玩家解锁圣遗物副本，在交互解锁传送之后它就被消费完毕了，里面的内容通常是没什么趣味的，这里不是说早期的一些一次性解谜和战斗秘境。基建相比之下复杂而有趣得多，并且它也不容易中断探索的心流，因为与基建产线的工作时间相比，早回和晚回基地的时间几乎不值一提，不会因为获得工业点数就直接断了整个探索体验。基建也并不需要玩家每天走过去刷本，只需要进行UI点击就能收菜，减少了枯燥的体验（当然，刷本的时间也没有被消除就是了），转化为设计基建产线的更有趣体验。

对于第三点，因为基建实际并不适合所有玩家游玩，因此终末地给出了蓝图的解法，可以摆放官方和其它玩家分享的蓝图复刻已经设计好的产线部分。在这种情形，一般会有攻略博主等KOL研究一个比较优化的产线蓝图放在社区里，同时玩家也能在评论区或者即时社交软件小规模地交换蓝图，交流设计产线的经验。因为基建的“抄作业”比起战斗抄轴更加简单，只是复制粘贴蓝图的操作，所以它应该是一个可以活跃社区的话题。玩家进行社交时会容易被社区吸引，会提高对整个游戏的粘性。


### 装置设计

因为这部分没什么意思，所以放在后面。

假如不想复刻一个一样的，也许不该把所有装置全列一遍。笔者之前写王国之泪分析时已经试过了，而后觉得效果不好又删掉，并不划算。因此这里只考虑某种IO上的模块类型，以输入和输出的个数来区分。


| 类型   | 例子        | 备注                                      |
| ---- | --------- | --------------------------------------- |
| 1进1出 | 传送带，准入口   | 传送带速率为0.5u/s                            |
| 1进3出 | 分流器       |                                         |
| 2进2出 | 物流桥       |                                         |
| 1出   | 取货口       | 下文记为`S(type): X`，其中`S`是函数名，`X`是产物类型     |
| 3进3出 | 精炼炉，粉碎机   | 下文记为`M3(x1, x2, x3): (Y1 * Y2 * Y3)`    |
| 5进5出 | 采种机和种植机   | 下文记为`M5(x1, x2 .. x5): (Y1 * Y2 .. Y5)` |
| 6进6出 | 研磨机，灌装机   | 下文记为`M6(x1, x2 .. x6): (Y1 * Y2 .. Y6)` |
| 3进   | 热能池，协议储存箱 | 下文记为`T(x1, x2, x3): !`                  |

装置俯瞰都是矩形。这样方便网格化，易于在UI里操作，不容易摆得太难看，程序上也储存简便。

一个没有环的流水线可以用函数组合的方式来表达，记号中的大写字母为物品类型和装置的函数名，小写字母为物品。`!`记号代表什么都没有且不能往下连接。为了方便，不使用的参数直接忽略，因为理想条件下顺序对机器不是那么重要，传送带也被省略掉。下文用这样的记号来表达装置。假如引入标识符（identifier），那么有环的流水线也能以此来表达。

作为例子，生产晶体外壳的流水线为`T(M3(S(源矿)))`，生产蓝铁装备原件的流水线为`T(M6(M3(S(源矿)), M3(S(蓝铁矿))))`，生产高晶装备元件的流水线为
```
a: 高晶粉末 = M6(M3(S(紫晶矿)), M3(S(砂叶)))
b: 致密晶体粉末 = M3(M6(M3(S(源矿)), M3(S(砂叶))))
return T(M6(M3(a), M3(B)))
```

大体来讲，里面有一个很明显的重复模式，即`M6(M3(..), M3(..))`，因为M6装置的宽度正好是M3装置的两倍，这样设计更好安排空间，结果也更好看。

种植机和采种机的地位是特殊的。一般由两个种植机和一个采种机相组合，以砂叶为例，形成
```
i: 砂叶
a1: 砂叶种子, a2: 砂叶种子 = M5(i, b)
b: 砂叶 = M5(a1)
return M5(a2): 砂叶
```
其中的a和b是互相连接的。

用同样的方式也可进行速度计算：

终末地所有的合成都类似于一种$$aX+bY=cZ$$或$$aX=cZ$$的形式，$$X,Y,Z$$是物品，$$a,b,c$$是个数。因此速率好求，为$$z=\mathrm{min}(\mathrm{min}(x/a,y/b)* c,d)$$或$$z=\mathrm{min}(x/a * c,d)$$，$$x,y,z$$为$$X,Y,Z$$的速率（不要和上面的代码里的记号弄混），d是机器自己的最大输出率。一个产线最终的速率都是嵌套的这种计算。

传统带拥有0.5u/s速度限制。不过机器的需求一般也只有不到0.5u/s。

举例：生产砂叶的速率：
```
在初始情况，i 的速率是正无穷
b 的速率是 0
因为i和b是一类物品，速度合并
a1的速率 + a2的速率 = min(正无穷, 0.5) * 2 = 1
因此a1的速率 = a2的速率 = 0.5
b的速率 = min(0.5, 0.5) = 0.5
在i消耗殆尽后速度为0，此时i+b速率仍>=0.5，a1和a2速率不变
输出速率为min(0.5, 0.5) = 0.5，生产砂叶的速率为0.5u/s
```

补充：在武陵推出了水管。水管可以做到2u/s，方便使用分流和汇流。

在游戏中，终末地提供了理论速率数据，实际上是假定输入不构成瓶颈，直接计算输出速率的总和，能帮助玩家判断自己的装置是否有拖累效率的部分。

### 问题
对于基础操作体验的角度：
- 对手机玩家不友好：手机的屏幕小，在铺设传送带时有困难。这只是困难的一例。
- 许多交互似乎缺乏打磨：例如，在不使用俯瞰视角时铺设水管比较困难，在基地外无法使用俯瞰视角。但是水泵一般必须在基地外，因此不得不在没有俯瞰视角时铺设水管，甚至还要再这种情况下使用分流器。

对于留存的角度：
- 高认知负荷，容易吓跑一部分玩家：需要用一些时间细心理解，有时玩家不愿意投入这样的时间。内置的教程对他们来说意义很小。~~我是来玩二游的，你们要干什么（指基建产物速率配平计算可能太像“做题”了）？~~。
- 没有长期游玩的情感上的回报：不像战斗那样有一个难以达到上限的外部指标量化自己做得是否好。战斗镀层时间竞速几乎永无止境，基建的产率因为各种因素有一个不太难达到的上限。而且复制别人的蓝图可以一键满速。
